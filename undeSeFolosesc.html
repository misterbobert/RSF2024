Advanced Encryption Standard (AES)

  • Este algoritmul standard folosit de Guvernul Statelor Unite ale Americii, de alte organizații și prezent în Arcserve Unified Data Protection (UDP) software
  • Vine în 3 forme, diferența dintre ele fiind dimensiunea cheii: în 128 bit, 192 bit, 256 bit
  • Este considerat un algoritm securizat împotriva tutoror atacurilor, cu excepția forței brute

Triple DES

  • Este algoritmul standard folosit de Guvernul Statelor Unite ale Americii, de alte organizații și prezent în Arcserve Unified Data Protection (UDP) software
  • Vine în 3 forme, diferența dintre ele fiind dimensiunea cheii: în 128 bit, 192 bit, 256 bit
  • Este considerat un algoritm securizat impotriva tutoror atacurilor, cu excepția forței brute

Caesar Cipher

  • Este una dintre cele mai simple metode de criptare, utilizând o deplasare fixă a literelor alfabetului
  • Vulnerabil la atacuri de forță brută din cauza numărului mic de posibile chei (26 în cazul alfabetului englez)
  • Este utilizat mai mult ca metodă educativă decât ca soluție practică modernă

HEXA

  • Codificare hexadecimală reprezintă date binare folosind cifrele 0-9 și literele A-F
  • Este folosită frecvent pentru a reprezenta chei de criptare, hash-uri și alte date
  • Nu este o metodă de criptare în sine, ci mai degrabă o metodă de codificare a informației

RSA Security

  • Este un algoritm de criptare asimetrică bazat pe factorizarea numerelor mari în factori primi
  • Cheia publică este utilizată pentru criptare, iar cheia privată pentru decriptare
  • Este considerat foarte sigur dacă sunt utilizate chei suficient de mari (2048 biți sau mai mari)

Blowfish

  • Algoritm de criptare simetrică dezvoltat ca o alternativă la DES
  • Utilizează o cheie variabilă între 32 și 448 biți, oferind un echilibru între securitate și performanță
  • Este popular pentru criptarea datelor și a parolelor, dar nu este la fel de larg utilizat ca AES

ECC (Elliptic Curve Cryptography)

  • Utilizează curbe eliptice pentru a oferi criptare asimetrică cu chei mult mai mici decât RSA
  • Este foarte eficientă și oferă aceeași securitate ca și RSA, dar cu dimensiuni mai mici ale cheilor
  • Este folosită frecvent în aplicații mobile și alte medii unde puterea de calcul este limitată

TwoFish

  • Algoritm de criptare simetrică dezvoltat ca succesor al Blowfish
  • Acceptă chei de până la 256 biți și este cunoscut pentru eficiența și flexibilitatea sa
  • Deși nu a fost selectat pentru AES, rămâne o opțiune solidă pentru criptarea datelor

Diffie-Hellman

  • Este un protocol de schimb de chei care permite două părți să genereze împreună o cheie secretă, utilizând canale publice
  • Este vulnerabil la atacuri de tip "man-in-the-middle" dacă nu este utilizat cu alte metode de autentificare
  • Rămâne o metodă fundamentală pentru securitatea internetului și a rețelelor

RC4

  • Este un algoritm de criptare simetrică de tip flux, cunoscut pentru viteza sa mare
  • A fost folosit în WEP și TLS, dar este considerat nesigur din cauza vulnerabilităților descoperite
  • Deși rapid, a fost înlocuit cu algoritmi mai siguri, cum ar fi AES

IDEA (International Data Encryption Algorithm)

  • Algoritm de criptare simetrică folosit în PGP (Pretty Good Privacy)
  • Utilizează o cheie de 128 de biți și este considerat sigur, deși mai puțin popular decât AES
  • Performanța este bună, dar este în mare parte înlocuit de algoritmi moderni

Paillier

  • Este un algoritm de criptare asimetrică bazat pe probleme matematice legate de factorizare
  • Oferă criptare omomorfă, permițând operarea pe date criptate
  • Este utilizat în aplicații ce necesită confidențialitate, cum ar fi votul electronic și calculul privat
